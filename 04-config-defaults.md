# 04 — Config & ค่า Default

> ตัวเลขทั้งหมดอยู่ใน **หน้า Config ของแอดมิน** ปรับเองได้ ไม่ต้องแก้โค้ด
> ค่าด้านล่างคือ **default เริ่มต้น (ข้อเสนอ)** — เอาไปจูนหน้างานตามโปรแต่ละสัปดาห์ได้
> ⚠️ ทุกค่าต้องผ่าน **validation** ก่อนบันทึก (กันค่าที่ทำให้เกมพัง)

---

## 1. ผู้เล่น & บทบาท
| ค่า | default | หมายเหตุ |
|-----|---------|---------|
| จำนวนผู้เล่น | 11 | ปรับได้ |
| จำนวนสปาย | 2 (A, B) | ปรับได้ |

## 2. เกณฑ์โหวตชนะ
| ค่า | default | หมายเหตุ |
|-----|---------|---------|
| สัดส่วนเกณฑ์ชนะ | **0.72 × คนที่มา** ปัด round | 11→8, 10→7, 9→6, 8→6 |
| ลดเกณฑ์ (ไอเทม) | −25% (× 0.75) ปัด round | ขั้นต่ำไม่ต่ำกว่า 2 |

## 3. ค่าเปิดโหวต (scale + สะสม)
| ค่า | default | หมายเหตุ |
|-----|---------|---------|
| ค่าเปิดฐาน (ต่อหัว) | 3 เหรียญ × คนที่มา | เช่น 11 คน = 33 เหรียญรวม |
| อัตราแพงขึ้นเมื่อไม่เปิด | +50% ต่อวันทำงานที่ข้าม | สะสมทบ จนกว่ามีคนเปิด แล้วรีเซ็ต |
| ตัวคูณจากกาชา | ×1.5 หรือ ×0.5 | ใช้กับ "ครั้งถัดไป" 1 ครั้ง |
| เปิดได้ต่อวัน | 1 ครั้ง | |
| คืนกรณีโหวตโดนบริสุทธิ์ | 1/4 ของกองที่จ่าย (ปัดลง) | จำนวนผู้รับ < คนที่มา |

## 3b. เบาะแสหลังโหวต (§4.5–4.6)
| ค่า | default | หมายเหตุ |
|-----|---------|---------|
| เกณฑ์ §4.5 ประกาศจำนวนสปาย | ผู้ถูกโหวต ≥ 4 คน | + ต้องมีสปายในกอง |
| เกณฑ์เบาะแส (ก) ดูคนถูกโหวต | ผู้ถูกโหวต ≥ 3 คน | ต่ำกว่านี้ = จ่ายฟรี |
| ราคาเบาะแส | 1/4 ของค่าเปิดโหวตรอบนี้ (ปัดลง) | งานกลุ่ม รวมเหรียญซื้อ |
| ลิมิตซื้อเบาะแส | **1 ครั้ง/รอบโหวต (ทั้งทีม)** | ผลเปิดเผยทุกคน |
| (ก) จำนวนรูปคนถูกโหวตที่โชว์ | 1 คน (สุ่ม) | |
| (ข) จำนวนรูปคนไม่ถูกโหวตที่โชว์ | สูงสุด 4 คน (สุ่ม) | มีน้อยกว่าก็โชว์เท่าที่มี |
| §4.5 ทำงานตอนเกราะพลิกผล | ทำงาน (ประกาศ) | สลับเป็น "ระงับ" ได้ |

## 4. โจทย์เชาว์ (มาจากกาชาเท่านั้น — ไม่มีขายในร้าน)
| ค่า | default |
|-----|---------|
| รางวัลตอบถูก (ผู้หมุน) | +10 เหรียญ |
| โทษตอบผิด (ทุกคน) | −3 เหรียญ/คน (ไม่มีก็ไม่คืน) |
| ที่มา | ผลกาชา "ได้โจทย์ฟรี" (น้ำหนัก 15% ดู §6) |

## 5. ร้านไอเทม (ไอเทมโหวต — ซื้อลับ)
| ไอเทม | ราคา default | ลิมิต/วัน |
|-------|------|----------|
| โหวต 2 เสียง | 10 | 1 |
| ลบ 1 เสียง | 8 | 1 |
| สลับผลโหวต | 12 | 1 |
| R = ลดเกณฑ์ 25% | 12 | 1 |
| P = ป้องกันการลดเกณฑ์ | 12 | 1 |
| รวมไอเทมโหวตถือพร้อมกัน | สูงสุด 2 ชิ้น/คน | กันสะสมเยอะเกิน (รวมที่ได้จากกาชา) |

**ค่ามินิเกมเกณฑ์ (R/P):**
| ค่า | default |
|-----|---------|
| R ลดเกณฑ์ | 25% |
| P อ่อนค่า R (เมื่อใช้หลัง) | เหลือ 12% |
| P ใช้ก่อน R | บล็อก (ลด 0%) |
| เกณฑ์หลังลด | ปัด round, ขั้นต่ำ ≥ 2 |

## 6. กาชา (หมวด B)
| ค่า | default |
|-----|---------|
| ราคาหมุน | 5 เหรียญ |
| ลิมิตต่อคน/วัน | 2 ครั้ง |

**ตารางน้ำหนักผล (รวม 100%) — ปรับได้:**
| ผล | น้ำหนัก |
|----|---------|
| ตัวเองได้เหรียญ (+8) | 13% |
| ตัวเองเสียเหรียญทั้งหมด | 9% |
| ทุกคนได้เหรียญ (+3) | 11% |
| ทุกคนที่มี < x(5) รับ y(5) | 9% |
| ทุกคนเสียเหรียญ (−3) | 9% |
| โหวตหน้า ×1.5 | 9% |
| โหวตหน้า ×0.5 | 8% |
| 🎁 ได้ไอเทมโหวตสุ่ม 1 ชิ้น | 9% |
| 🧠 ได้โจทย์เชาว์ฟรี 1 ข้อ | **15%** |
| 🛡️ สปาย A/B ได้เกราะ | 8% |

> **ผลสำรอง (fallback) เมื่อช่องที่ออกใช้ไม่ได้:**
> - "เกราะ" แต่สปายมีเกราะครบแล้ว → ตกเป็น "ทุกคนได้เหรียญ"
> - "ได้ไอเทม" แต่กระเป๋าผู้หมุนเต็ม (ถือ 2 ชิ้นแล้ว) → ตกเป็น "ตัวเองได้เหรียญ"

## 6b. ไอเทม & เกราะ — การคงอยู่
| ค่า | default |
|-----|---------|
| สุ่มสปายใหม่ → ไอเทมทั่วไป | **ติดตัวคน** (ไม่รีเซ็ต อาจเสียฟรีถ้าเป็นคนปกติ) |
| สุ่มสปายใหม่ → เกราะ (ถ้ายังไม่ถูกใช้) | **ย้ายไปกับสปายคนใหม่ของ slot เดิม** (เกราะมีชิ้นเดียว ผูก slot A/B) |
| เกราะถูกใช้แล้ว ก่อนสุ่มใหม่ | หายจากเกม สปายใหม่ไม่มีเกราะ |
| ไอเทมจากกาชา | ประกาศเปิดเผยว่าใครได้อะไร |
| ไอเทมจากร้าน (หมวด C) | ซื้อลับ ไม่เปิดเผย |

## 7. เวลา
| ค่า | default |
|-----|---------|
| วันเริ่ม | จันทร์ |
| วันจบ | จันทร์ถัดไป (บ่าย) |
| วันหยุด | เสาร์-อาทิตย์ (หยุดทุกอย่าง ค่าโหวตไม่ขยับ) |

---

## 8. Validation Rules (กันค่าพัง)
- เกณฑ์ชนะที่คำนวณได้ต้อง **≥ 2 และ ≤ คนที่มา**
- จำนวนสปาย **< จำนวนผู้เล่น** (และ ≥ 1)
- ราคา/รางวัล/โทษ ต้อง ≥ 0
- น้ำหนักกาชารวม = 100% (auto-normalize ถ้าไม่ครบ)
- 1/4 refund ต้องปัดให้ **จำนวนผู้รับ < คนที่มา** เสมอ (คงกติกา "มีคนไม่ได้คืน")
