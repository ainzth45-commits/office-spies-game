> แผนนี้แตกงานเป็น **โมดูลย่อยที่สร้าง/ทดสอบแยกได้** แต่ละโมดูลมี: หน้าที่ · input/output · เกณฑ์ผ่าน (done)
> ลำดับเรียงตาม dependency — ทำจากบนลงล่างได้เลย

## โมดูล 0 — Project Skeleton & State Core
**หน้าที่:** โครงแอป + data model + persist
- ตั้งโปรเจกต์ SPA, ล็อก landscape, หน้า Boot/Unlock (ปลดล็อก audio + load state)
- **Data model** (object เดียวเก็บทั้งเกม):
  ```
  GameState {
    config,                 // ค่าจาก config
    players[] { id, name, avatar },
    presentToday[],         // คนที่มาวันนี้
    roles { playerId: 'normal' | 'spyA' | 'spyB' },
    inventories { playerId: [voteItems] },   // กระเป๋าลับ
    shields { spyA: bool, spyB: bool },
    day,                    // 1..N (จ.–จ.หน้า)
    voteCostState { base, accumulatedMultiplier, openedToday },
    nextVoteMultiplier,     // จากกาชา ×1.5/×0.5
    history[],              // log ทุก action (ไว้ undo/ตรวจ)
    phase                   // setup | playing | guessing | ended
  }
  ```
- persist → localStorage ทุกครั้งที่ state เปลี่ยน + export/import JSON
- **Done เมื่อ:** เปิดบน iPad แนวนอน, สร้าง/แก้ state, refresh แล้วข้อมูลอยู่ครบ, export/import ได้

## โมดูล 4 — Vote Engine (แกนคำนวณ) ⭐ *สำคัญสุด*
**หน้าที่:** ฟังก์ชันบริสุทธิ์คำนวณผลโหวต — แยกออกจาก UI เพื่อเทสได้
- **Input:** รายการโหวต (voterId → targetId), ไอเทมที่ใช้ **เรียงตาม turn order** (double/remove/swap/R/P + เป้า), คนที่มา, เกราะ
- **มินิเกมเกณฑ์ R/P:** ประมวลตามลำดับ turn order ตาม [01 §7 ข้อ 4] — R โดนบล็อกถ้ามีหมวดเกณฑ์ถูกใช้ก่อน, P ก่อน=บล็อก/P หลัง=อ่อนเหลือ 12%, ตัวที่ 2 ของแต่ละชนิด=เสียฟรี
- **Process:** ตามลำดับใน [01 §7] เป๊ะๆ
  ```
  1. tally ดิบ (รวม double vote)
  2. apply remove-vote
  3. apply swap (2 คน)
  4. threshold = round(0.72 × present); ถ้ามี reduce → round(threshold × 0.75), floor ≥ 2
  5. หา winner (คนเดียวที่ ≥ threshold) → win/lose
  6. ถ้า win & winner = spy มีเกราะ → flip เป็น lose, consume shield
  ```
- **Output:** { result, winnerId?, winnerIsSpy?, shieldTriggered?, **votedPool[]** (คนที่ได้เสียง≥1 หลังปรับไอเทม), **notVotedPool[]**, **spiesInPoolCount** } — **ไม่คืนตัวเลขเสียงรายคน**
  - ⚠️ `shieldTriggered` ใช้ **ภายใน backend เท่านั้น** (consume เกราะ + บันทึก log) — **ห้ามแสดงผล UI/เสียง/ข้อความใดๆ** ผลที่โชว์ต้องเหมือน "แพ้ปกติ" 100% (ดู [01 §6.4])
- **Unit tests (บังคับ):** เสียงเท่ากันหลายคน, ลบเสียงจนเหลือ 0 (หลุดจาก votedPool), สลับกับตัวเอง, ลดเกณฑ์จนชน floor, เกราะพลิกผล, ครบ/ไม่ครบเกณฑ์, **นับ votedPool/spiesInPool ถูกเมื่อ ≥4 และ <4**, **R/P ครบทุกเคส** (R เดี่ยว, P-ก่อน-R บล็อก, R-ก่อน-P อ่อนเป็น 12%, R ตัวที่2, P ตัวที่2, P เดี่ยวไม่มี R)
- **Done เมื่อ:** เทสทุกเคสผ่าน (แปะ output)

## โมดูล 5 — Vote Flow UI ⭐
**หน้าที่:** ห่อ Vote Engine ด้วย UI
- เปิดโหวต: คำนวณค่าเหรียญ (base × present × accumulated × gachaMult) → ยืนยันจ่าย → mark openedToday + รีเซ็ต accumulated
- ลงคะแนนลับ: วนทีละคน (เลือกเป้า + ใช้ไอเทมจากกระเป๋า) → ม่านส่งเครื่อง → ติ๊กครบ
- เรียก Vote Engine → ประกาศผล (ดราม่า ไม่โชว์ตัวเลข)
- routing ผล: แพ้ / ชนะ-บริสุทธิ์ (→ โมดูล 9 หน้าจัดสรร) / ชนะ-สปาย (→ โมดูล 6) / เกราะพลิก
- **§4.5 ประกาศจำนวนสปายในกอง**: ถ้า votedPool ≥ 4 และ spiesInPoolCount ≥ 1 → ประกาศ "S จาก T"
- **§4.6 ช่วงซื้อเบาะแส (งานกลุ่ม)**: ทีมรวมเหรียญจ่าย 1/4 ค่าโหวต → เลือก (ก)/(ข) → สุ่มจาก votedPool/notVotedPool → **โชว์การ์ดรูปบนจอใหญ่ เห็นทั้งทีม** · **1 ครั้ง/รอบโหวต** · (ก) ทริกเมื่อ **≥3 คน**, <3 = จ่ายฟรี · (§4.5 ใช้ ≥4 แยกกัน)
- **Done เมื่อ:** เปิด-โหวต-ประกาศครบ, ค่าโหวตสะสม/รีเซ็ตถูก, ไอเทมถูกใช้และหักจากกระเป๋า, §4.5/§4.6 ทำงานถูกตามเงื่อนไข ≥4/<4 และมี/ไม่มีสปายในกอง

## โมดูล 6 — Guess Second Spy & Re-randomize
**หน้าที่:** ทายสปายคนที่ 2
- เลือก 1 ชื่อ (ตอบครั้งเดียว) → ถูก = ทีมชนะ (→ End) / ผิด = สุ่มสปายใหม่ → กลับโมดูล 3
- **การคงอยู่ตอนสุ่มใหม่ (ดู [01 §6.5]):**
  - ไอเทมทั่วไป (โหวต2/ลบ/สลับ/R/P) → **ติดตัวคนเดิม ไม่รีเซ็ต**
  - เกราะ (ถ้ายังไม่ถูกใช้) → **ย้ายไปให้สปายคนใหม่ของ slot เดิม** (เกราะชิ้นเดียว ผูก slot A/B); ถ้าถูกใช้แล้ว = หายจากเกม
- **Done เมื่อ:** ถูก→จบ, ผิด→สุ่มใหม่+ดู role ใหม่, ไอเทมติดตัวคนถูก, เกราะที่ยังไม่ใช้ย้าย slot ถูก, เกราะที่ใช้แล้วไม่ฟื้น

## โมดูล 7 — Quiz Flow (เรียกจากกาชาเท่านั้น)
**หน้าที่:** flow โจทย์ 2 ตัวเลือก — **ไม่มีหน้าซื้อในร้าน** ถูกเรียกโดยผลกาชา "ได้โจทย์ฟรี"
- โหลดคลังโจทย์จาก [07-quiz-bank.md] (โครง `{id,question,choiceA,choiceB,answer}`) — สุ่มข้อที่ยังไม่ออก (กันซ้ำ)
- โชว์โจทย์ → ทีมปรึกษา → ผู้หมุนกดตอบ → เฉลย → ประกาศผล (ผู้หมุนได้ +10 / ทุกคนเสีย −3)
- **Done เมื่อ:** สุ่มไม่ซ้ำในเกมเดียว, เฉลย/ผลถูกประกาศ, เชื่อมกับโมดูล 9 ได้

## โมดูล 8 — ร้านไอเทม: ไอเทมโหวตลับ + กระเป๋า
**หน้าที่:** ซื้อไอเทมโหวตแบบลับ (โหวต2/ลบ/สลับ/R/P)
- เลือกผู้ซื้อ → หน้าซื้อส่วนตัว (ลับ) → เก็บเข้า inventory ของคนนั้น → ม่านส่งเครื่อง
- เคารพลิมิตถือพร้อมกัน (สูงสุด 2/คน)
- **Done เมื่อ:** ซื้อแล้วเข้ากระเป๋าถูกคน, คนอื่นไม่เห็นว่าซื้ออะไร, ใช้ได้จริงในโมดูล 5

## โมดูล 9 — Gacha (10 ผล) + Spy Shield
**หน้าที่:** กาชาผลกระทบหมู่ + แจกไอเทม/โจทย์ + เกราะ
- ตู้กาชา + สุ่มตามน้ำหนัก [04 §6] (10 ผล) + animation/เสียง
- ประกาศผลเปิดเผย; เกราะ → บันทึก (shields[slot]=true, ผูก slot) + ประกาศ slot A/B
- **🎁 ได้ไอเทมโหวตสุ่ม** → ใส่กระเป๋าผู้หมุน (ประกาศชนิด) — fallback ถ้ากระเป๋าเต็ม
- **🧠 ได้โจทย์เชาว์ฟรี** → เรียกใช้ flow โมดูล 7 ทันที (ไม่กินลิมิตรายวัน, ไม่ต้องจ่าย)
- ตัวคูณโหวต ×1.5/×0.5 → set nextVoteMultiplier
- fallback: สปายมีเกราะครบแล้ว → ตกเป็น "ทุกคนได้เหรียญ"
- **Done เมื่อ:** สุ่มตามน้ำหนัก (รวม 100%), เกราะผูก slot + Vote Engine พลิกผลได้, แจกไอเทม/โจทย์ทำงาน, ตัวคูณมีผลรอบถัดไป, fallback ครบ

## โมดูล 10 — Day/Timeline & State Dashboard
**หน้าที่:** จัดการวัน + แดชบอร์ด

- "เลื่อนไปวันถัดไป": ถ้าไม่เปิดโหวต → accumulated +50% (เฉพาะวันทำงาน), รีเซ็ต openedToday, ล้างลิมิตซื้อรายวัน
- เดดไลน์ถึง + ยังไม่จับครบ → สปายชนะ
- **Done เมื่อ:** เดินวันถูก, ค่าโหวตสะสมถูก, เดดไลน์ทริกเกอร์ผลถูก
