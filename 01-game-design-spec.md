# 01 — สเปกเกมฉบับเต็ม: "สายลับในออฟฟิศ"

> เอกสารนี้คือ **กติกาแกนของเกม** ทุกตัวเลขที่เขียน `⚙️` = ค่า default ปรับได้ในหน้า config (ดู [04-config-defaults.md](04-config-defaults.md))

---

## 1. ภาพรวม

| หัวข้อ | รายละเอียด |
|--------|-----------|
| ประเภท | สืบสวนซ่อนบทบาท (social deduction) แบบเล่นยาวข้ามวัน |
| ผู้เล่น | ~11 คน `⚙️` (รองรับปรับจำนวนได้) |
| สายลับ | 2 คน — **สปาย A** และ **สปาย B** |
| ระยะเวลา | เริ่มจันทร์ → จบจันทร์หน้าตอนบ่าย ข้ามเสาร์-อาทิตย์ (~6 วันเล่นจริง) |
| อุปกรณ์ | iPad Air 5 แนวนอน เครื่องเดียว (แอดมินคุม) |
| สกุลเงิน | เหรียญจริง รับ/คืนที่ "ซุป" — แอปไม่นับยอดรายคน |

**เป้าหมาย:**
- 👥 **ฝ่ายผู้เล่น (ทีม)** — จับสายลับให้ครบ **ทั้ง 2 คน** ก่อนหมดสัปดาห์
- 🕵️ **ฝ่ายสายลับ** — เอาตัวรอดถึงจันทร์หน้าบ่าย + ปั่นให้ทีมเสียเหรียญ/เข้าใจผิด

## 2. บทบาท (Roles)

### 2.1 ผู้เล่นปกติ (~9 คน)
- เปิดดู role เห็นข้อความ: **"คุณคือผู้เล่นปกติ"**
- เป้าหมาย: หาสายลับ ใช้เหรียญอย่างชาญฉลาด

### 2.2 สปาย A / สปาย B (2 คน)
- เปิดดู role เห็น: **"คุณคือสายลับ [A/B]"** + **เห็นว่าใครคือคู่หู** (รูป/ชื่อสปายอีกคน)
- สปาย 2 คนรู้จักกัน → ร่วมมือลับ: ไม่โหวตกันเอง, ช่วยกันปั่น, แอบซื้อไอเทมโหวตเสริมกัน
- ป้ายกำกับ A/B สำคัญ: ใช้ตอนกาชาออกเกราะ (ระบบประกาศว่า "สปาย A ได้เกราะ")

### 2.3 การดู role
- ผู้เล่นเดินมาที่ iPad **ทีละคน** กดเปิดดู role ของตัวเอง → ดูเสร็จกดปิด (มีหน้าม่านปิดกั้นสายตาคนถัดไป)
- เกิดขึ้น 2 จังหวะ: **(1) เริ่มเกม** และ **(2) ทุกครั้งที่สุ่มสปายใหม่**
- ทุกคนเดินมาดูเหมือนกันหมด → ไม่มีใครรู้ว่าใครเพิ่งกลายเป็นสปาย (ข้อมูลไม่รั่ว)

## 3. เหรียญ & เศรษฐกิจ

- **เหรียญเป็นของจริงทั้งหมด** จัดการที่ซุป (แจกตามโปรรายวัน / เก็บคืน / วนแจกใหม่)
- **แอปไม่เก็บยอดเหรียญรายคน** — ทำหน้าที่: เก็บ role, ถือกระเป๋าไอเทมลับรายคน, รันโหวต, สุ่มกาชา/โจทย์, ประกาศผล
- เคสที่ต้องรู้ยอด (เช่น "ใครมีเหรียญ < x") → **ซุปเช็คเหรียญจริงหน้างาน** (ตั้งใจให้เกิดปฏิสัมพันธ์)
- เหรียญที่ "หาย" (เปิดโหวตแพ้, โดนปรับ) → กลับไปที่ซุปเพื่อ **วนแจกใหม่** (เหรียญหมุนในระบบ ไม่งอกไม่เฟ้อ)

## 4. ระบบโหวตกำจัด ⭐ (หัวใจเกม)

### 4.1 การเปิดโหวต
- ใครก็ **เสนอเปิดโหวต** ได้ → ทุกคนต้อง **รวมเหรียญ** ตามจำนวนที่กำหนด (แพง) จ่ายที่ซุป
- เปิดได้ **วันละ 1 ครั้งเท่านั้น** `⚙️`
- **ถ้าวันไหนไม่เปิดโหวต → ค่าเปิดโหวตครั้งถัดไปแพงขึ้น สะสมไปเรื่อยๆ** `⚙️` จนกว่าจะมีคนเปิด แล้ว **รีเซ็ตกลับค่าฐาน** (เสาร์-อาทิตย์ไม่นับ ไม่ทำให้แพงขึ้น)
- 🕵️ ชั้นเชิงสปาย: สปายอาจไม่ยอมจ่าย (ดูน่าสงสัย) หรือยุให้เปิดโหวตบ่อยๆ เพื่อผลาญเหรียญทุกคน

### 4.2 การลงคะแนน
- เปิดหน้าโหวตบน iPad → **ห้ามปรึกษากันระหว่างโหวต**
- ผู้เล่นผลัดกันเดินมากดเลือกที่ iPad **แบบลับ** (คนอื่นไม่เห็นว่าใครเลือกใคร) — คุยกันทีหลังได้
- ระหว่างเลือก ผู้เล่นที่ถือ **ไอเทมโหวตลับ** สามารถเลือกใช้ได้ในช่องส่วนตัวของตัวเอง (ดูข้อ 6.3)

### 4.3 การประกาศผล — **ไม่เปิดเผยจำนวนเสียง**
ผลมี 2 แบบ:

**🔴 โหวตแพ้** = ไม่มีใครได้เสียงถึงเกณฑ์
- ไม่เกิดอะไรขึ้น เหรียญที่จ่ายเปิดโหวต **หายฟรี** (คืนซุป)
- (ตั้งใจให้แพ้บ่อย เพราะแจกเหรียญง่าย เหรียญต้องหมุนออก)

**🟢 โหวตชนะ** = มีคน **คนเดียว** ได้เสียง **ถึงเกณฑ์ `⚙️`**
แยกเป็น 2 กรณีตามว่าคนที่โดนมากสุดเป็นใคร:

  **(ก) คนนั้น = สายลับ**
  - ทีมได้สิทธิ์ **ทายสายลับอีกคน** (ปรึกษากันได้ ตอบรวมเป็นทีม **1 ครั้งเท่านั้น**)
    - **ทายถูก** → 🎉 ทีมชนะ เกมจบ
    - **ทายผิด** → ระบบ **สุ่มสปายใหม่** (คนเดิมอาจเป็นอีกหรือเปลี่ยนก็ได้) → ทุกคนเดินมาดู role ใหม่ → เล่นต่อ
  - เหรียญเปิดโหวต = หายไป (คืนซุป) ไม่มีการคืน

  **(ข) คนนั้น = ผู้เล่นบริสุทธิ์**
  - คืนเหรียญ **1/4 ของกองที่จ่าย** `⚙️` ให้ทีม
  - **จำนวนที่คืน < จำนวนคน** → มีคนไม่ได้คืนแน่นอน → ทีมต้องตัดสินใจว่าใครไม่ได้
  - ทีมเลือก **1 คนเป็นผู้ถือเหรียญ** → คนนั้นมีสิทธิ์ตัดสินว่าใครได้เท่าไหร่
  - 🕵️ ชั้นเชิงสปาย: สปายอาจเสนอตัวเป็นคนถือ เพื่อจัดสรรเหรียญเข้าข้างตัวเอง/พวก

### 4.4 เกณฑ์ชนะ (scale ตามคนที่มา)
- เกณฑ์ = **round(0.72 × จำนวนคนที่มาวันนั้น)** `⚙️`
  - 11 คน → 8 · 10 คน → 7 · 9 คน → 6 · 8 คน → 6
- **คนลาวันนั้น**: ไม่ถูกนับในเกณฑ์ และถ้าทีมชนะ คนลา **ไม่ได้รับส่วนแบ่ง**
- ค่าเปิดโหวตก็ scale ตามจำนวนคนที่มาเช่นกัน `⚙️`

### 4.5 🔎 ประกาศจำนวนสปายในกองผู้ถูกโหวต (Spy-in-pool reveal)
ทำงาน **หลังประกาศผลโหวต** (ทั้งกรณีแพ้และชนะ) เป็นเบาะแสให้ทีมจำกัดวง:
- **เงื่อนไขทริกเกอร์:** มี **ผู้ถูกโหวต ≥ 4 คน** (คนที่ได้เสียง ≥ 1 ในผลที่ปรับไอเทมแล้ว) **และ** ในกองนั้นมีสปายจริงอย่างน้อย 1 คน
- ระบบประกาศ: *"รอบนี้มีสายลับถูกโหวต **S** คน จากผู้ถูกโหวตทั้งหมด **T** คน"* (เช่น 2 จาก 6)
- **ไม่บอกว่าใคร / ไม่บอกจำนวนเสียง** → ผู้เล่นต้องไปนั่งถามกันเองว่า "เมื่อกี้โหวตให้ใคร" เพื่อจำกัดวง
- ถ้าในกองผู้ถูกโหวต**ไม่มีสปายเลย** → ระบบเงียบ ไม่พูดอะไร
- 🕵️ จุดน่าสนใจ: สปายที่ใช้ไอเทม "ลบเสียง" ลบเสียงตัวเองจนเหลือ 0 จะ**หลุดจากกองผู้ถูกโหวต** → ไม่ถูกนับในตัวเลขนี้ (เนียน)
- ⚠️ ปฏิสัมพันธ์กับเกราะ: ถ้าเกราะพลิกผล "ชนะ→แพ้" แต่ข้อ 4.5 ยังประกาศว่ามีสปายในกอง → ทีมอาจเดาได้ว่ามีเกราะ (ดู [§7 หมายเหตุ] — เลือกได้ว่าจะให้ 4.5 ทำงานตอนเกราะพลิกหรือไม่)

### 4.6 🃏 ซื้อเบาะแสหลังโหวต (Post-vote Clue) — **งานกลุ่ม**
หลังข้อ 4.5 เปิด **ช่วงซื้อเบาะแส** — เป็น **งานกลุ่ม เหมือนการเปิดโหวต** (ทีมรวมเหรียญกันซื้อ):
- **ราคา = 1/4 ของค่าเปิดโหวตรอบนี้** `⚙️` (รวมเหรียญจ่ายที่ซุป)
- **ลิมิต:** **1 ครั้งต่อรอบโหวต** (ทั้งทีม ไม่ใช่รายคน)
- **การมองเห็น:** เบาะแส **เปิดเผยให้ทุกคนเห็น** (เพราะซื้อด้วยกัน) — โชว์จอใหญ่
- ทีมตกลงกันว่าจะซื้อไหม + เลือก **1 ใน 2**:

  **(ก) ขอดูรูป "คนที่ถูกโหวต" รอบนี้**
  - ถ้าผู้ถูกโหวต **≥ 3 คน** `⚙️` → สุ่มโชว์รูป **1 คน** จากกองผู้ถูกโหวต
  - ถ้าผู้ถูกโหวต **< 3 คน** → **จ่ายเหรียญฟรี** (ไม่ได้อะไร — เป็นการเสี่ยงดวง)
  - *(เกณฑ์ 3 แยกจาก §4.5 ที่ใช้ ≥4 — ตั้งใจให้ (ก) ทริกง่ายขึ้น เพิ่มความเร้าใจ + โอกาสปั่นกระแส)*

  **(ข) ขอดูรูป "คนที่ไม่ถูกโหวต" รอบนี้**
  - ระบบสุ่มคนที่**ไม่ได้ถูกโหวต**รอบนี้ มาโชว์ **สูงสุด 4 คน** (มีน้อยกว่า 4 ก็โชว์เท่าที่มี)

- 🧠 คุณค่า: เบาะแสบอก "ใครอยู่/ไม่อยู่ในกองผู้ถูกโหวต" เอาไปประกอบกับ §4.5 เพื่อไล่หาสปาย (ไม่ได้บอกว่าใครเป็นสปายตรงๆ)
- 📌 "กองผู้ถูกโหวต" คิด**หลังคำนวณไอเทม/เอฟเฟกต์ทุกอย่างแล้ว** (ตรงกับ §7 ข้อ 7)

## 6. ร้านค้า & กาชา

เกมมี 2 ช่องทางใช้เหรียญ: **ร้านไอเทม** (ซื้อของที่ตั้งใจอยากได้) และ **กาชา** (สุ่มลุ้น)
ผู้เล่นเดินมาที่ iPad (จ่ายเหรียญจริงที่ซุป แอดมินกดบันทึกในแอป)

### 6.1 🧠 โจทย์เชาว์ — **มาจากกาชาเท่านั้น (ไม่มีขายในร้าน)**
> เดิมเคยเป็นของซื้อ แต่**ตัดออกจากร้าน** เพราะไม่มีใครยอมจ่ายซื้อโจทย์ที่เสี่ยงพาทีมซวย — ย้ายไปเป็นผล**กาชา** (คนแห่มาหมุนมากกว่า + ได้ฟรี ฮากว่า)
- เมื่อกาชาออกผล "ได้โจทย์เชาว์" → เปิด **โจทย์ 2 ตัวเลือก** (เชาว์/คณิต ดูคลังใน [07-quiz-bank.md]) ทันที
- ทั้งทีมปรึกษากันได้ → ผู้หมุนเป็นคนกดตอบ
- **ตอบถูก** → ผู้หมุนได้เหรียญเพิ่ม `⚙️`
- **ตอบผิด** → **ทุกคนเสียเหรียญคืนซุป** `⚙️` (ใครไม่มีก็ไม่ต้องคืน)
- 🎭 ความเสี่ยงตกที่ทุกคน แต่รางวัลตกที่ผู้หมุน → ลุ้น/ฮา และเกิดดราม่าได้

### 6.2 🎰 กาชา (ผลกระทบหมู่ เปิดเผย)
หมุนแล้วสุ่มผล ประกาศให้ทุกคนเห็น ตารางผล (น้ำหนักความน่าจะเป็น `⚙️`):

| ผล | ผลกระทบ |
|----|---------|
| ตัวเองได้เหรียญ | ผู้หมุนรับเหรียญ `⚙️` |
| ตัวเองเสียเหรียญทั้งหมด | ผู้หมุนคืนเหรียญทั้งหมดที่ซุป |
| ทุกคนได้เหรียญ | ทุกคนรับเพิ่ม `⚙️` |
| ทุกคนที่มี < x รับ y | ซุปเช็คหน้างาน คนเหรียญน้อยได้เพิ่ม |
| ทุกคนเสียเหรียญ | ทุกคนคืน `⚙️` |
| โหวตครั้งหน้า ×1.5 | ค่าเปิดโหวตถัดไปแพงขึ้น |
| โหวตครั้งหน้า ×0.5 | ค่าเปิดโหวตถัดไปถูกลง |
| **🎁 ได้รับไอเทมโหวตสุ่ม 1 ชิ้น** | สุ่มจากหมวด C (โหวต2/ลบ/สลับ/R/P) เข้ากระเป๋าผู้หมุน — **ประกาศเปิดเผยว่าได้อะไร** |
| **🧠 ได้รับโจทย์เชาว์ 1 ข้อ (ฟรี)** | เปิดโจทย์หมวด A ทันที (ไม่ต้องจ่าย) — ตอบถูกผู้หมุนได้เหรียญ/ผิดทุกคนเสีย · **ไม่กินลิมิตซื้อหมวด A รายวัน** |
| **🛡️ สปาย A/B ได้เกราะ** | **แจ็คพอตสายลับ** (ดู 6.4) — ประกาศ slot (A/B) แต่ไม่บอกตัวคน |

> **🎁 ไอเทมจากกาชา ≠ ไอเทมจากร้าน:**
> - **ร้าน (หมวด C)** = ซื้อ**ลับ** ไม่มีใครรู้ว่าได้อะไร
> - **กาชา** = ถูกกว่าแต่สุ่ม และ **ทุกคนรู้ว่าใครได้ไอเทมอะไร** (จะใช้ไหมอีกเรื่อง)
> - 🎭 ผลพลอยได้: **ผู้เล่นปกติอาจได้ไอเทมสายสปาย (เช่น P)** → ใช้แล้วเกิดผล อาจถูกเพื่อนสงสัย / หรือถ้าถูกสุ่มเป็นสปายทีหลังก็เข้าทางพอดี
> - ⚠️ ถ้ากระเป๋าเต็ม (เกินลิมิตถือพร้อมกัน) ตอนกาชาแจกไอเทม → ใช้ผลสำรอง (ดู config)

### 6.3 🛒 ร้านไอเทม — ไอเทมสายโหวต (ซื้อลับ ใช้ลับ)
- **ซื้อแบบความลับ**: คนอื่นรู้แค่ "ใครเดินไปซื้อ" ไม่รู้ว่าซื้ออะไร
- เก็บในกระเป๋าลับรายคน → **ใช้ได้เฉพาะตอนโหวต** (ในช่องส่วนตัวที่ iPad)

| ไอเทม | ผล |
|-------|-----|
| 🗳️ โหวต 2 เสียง | คะแนนของผู้ใช้นับเป็น 2 |
| ➖ ลบ 1 เสียง | ลบ 1 เสียงออกจากเป้าที่เลือก (ไม่มีใครรู้ว่าใครมีกี่เสียง) |
| 🔄 สลับผลโหวต | เลือก 2 คน สลับจำนวนเสียงกัน |
| 📉 **R = ลดเกณฑ์ชนะ 25%** | เกณฑ์ชนะของโหวตนี้ลดลง 25% `⚙️` (โหวตชนะง่ายขึ้น) — *ทีมมักใช้* |
| 🛡️ **P = ป้องกันการลดเกณฑ์** | กัน R — *สปายมักใช้* (ดูกฎ "มินิเกมเกณฑ์" ด้านล่าง) |

- 🕵️ ของเล่นชั้นเชิงสปาย: แอบซื้อลบเสียง/สลับผลปกป้องตัวเอง

#### 🧩 มินิเกม "เกณฑ์" — กฎ R ปะทะ P (สำคัญตอนสร้าง)
ไอเทม **หมวดเกณฑ์** มี 2 ตัว: **R (ลดเกณฑ์)** และ **P (ป้องกันการลด)** "ก่อน/หลัง" = **ลำดับที่เดินมากดใช้ตอนโหวต**

| สถานการณ์ | % ที่เกณฑ์ถูกลด |
|-----------|----------------|
| ไม่มีใครใช้ R | 0% (P ที่ซื้อมาเสียเปล่า) |
| R อย่างเดียว | **−25%** `⚙️` |
| P ใช้ **ก่อน** R | R ถูกบล็อก → **0%** |
| R มาก่อน, P ใช้ **หลัง** | อ่อนลงเหลือ **−12%** `⚙️` |

- **มีผลแค่ "อันแรก" ของแต่ละชนิด**:
  - **R ตัวที่ 2** หรือ **R ที่กดหลังจากมีหมวดเกณฑ์ใดถูกใช้แล้ว (R หรือ P)** → **เสียฟรี/ถูกบล็อก** + ระบบเด้งข้อความ *"มีคนใช้ไอเทมหมวดเกณฑ์ไปแล้ว"* (ไม่บอกว่าเป็น R หรือ P)
  - **P ตัวที่ 2** → เสียฟรี + เด้งข้อความเดียวกัน *(P ซ้ำกับ P เท่านั้น — P ที่กดหลัง R ถือว่าใช้ได้ ทำหน้าที่ "อ่อนค่า")*
- 🤝 เพราะข้อความไม่บอกชนิด → **สปาย 2 คนต้องคุยกันเอง** ว่าใครถือ P จะได้ไม่กดซ้ำเปลืองฟรี

### 6.4 🛡️ เกราะแจ็คพอตสายลับ (Spy Shield)
- **มาจากกาชาเท่านั้น** + **มีได้ชิ้นเดียวในเกม** ผูกกับ **slot A หรือ B** (สุ่มตอนออก) — ระบบ **ประกาศ "สปาย A (หรือ B) ได้รับเกราะป้องกัน 1 ครั้ง"**
  - ทุกคนรู้ว่ามีเกราะ + รู้ว่าเป็น slot ไหน แต่ **ไม่รู้ว่าใครคือ A/B** → หวาดระแวงทั้งทีม
  - 🔗 เกราะ **ผูกกับ slot ไม่ใช่ตัวคน** → ถ้ายังไม่ถูกใช้แล้วสุ่มสปายใหม่ จะย้ายไปกับสปายคนใหม่ของ slot นั้น (ดู §6.5)
- **ทำงานอัตโนมัติ**: ถ้าโหวต "ชนะ" และคนโดนมากสุด = สปายที่ถือเกราะ → **พลิกผลเป็น "โหวตแพ้" ทันที** (ไม่ประกาศว่าชนะ) → ทีมสับสน นึกว่าเดาผิด ทั้งที่จับถูกตัว
- ใช้ได้ 1 ครั้งแล้วหมด
- 🤫 **เงียบสนิทตลอดกาล**: ระบบ **ไม่ประกาศ** ตอนเกราะทำงาน — และ **ตัวสปายเองก็ไม่รู้ว่าเกราะถูกใช้ไปแล้ว** (เพราะผลโหวตเป็นความลับ)
  - สปายรู้แค่ตอนได้รับเกราะ → ระหว่างนั้น**ชะล่าใจได้** (นึกว่ายังมีเกราะ)
  - จะรู้ว่าเกราะหมดก็ต่อเมื่อ **เปิดดูไอเทมตัวเองในรอบโหวตถัดไป** แล้วเห็นว่าเกราะหายไปแล้ว
  - ดังนั้น "ใช้ทิ้ง" เกิดขึ้นเงียบๆ ใน backend ไม่มี UI/เสียง/ข้อความใดๆ บอก ณ จังหวะนั้น

### 6.5 🎒 การคงอยู่ของไอเทม/เกราะ เมื่อสุ่มสปายใหม่ — **มี 2 กฎต่างกัน**

**(1) ไอเทมทั่วไป (โหวต2/ลบ/สลับ/R/P) → ติดตัว "คน"**
- สุ่มสปายใหม่แล้ว **ไม่รีเซ็ต** ยังอยู่กับคนเดิม
- สปายที่กั๊กไอเทมไว้ พอถูกสุ่มเป็น **ผู้เล่นปกติ** → ของที่ถืออาจไร้ประโยชน์ = **เสียเหรียญฟรี**
- ผู้เล่นปกติที่บังเอิญถือไอเทมสายสปาย พอถูกสุ่มเป็นสปาย → **เข้าทางทันที**

**(2) 🛡️ เกราะ → ติด "ตำแหน่งสปาย A/B" ไม่ใช่ตัวคน**
- ทั้งเกมมี **เกราะชิ้นเดียว** ผูกกับ slot ใด slot หนึ่ง (A หรือ B แบบสุ่มตอนกาชาออก)
- ถ้าเกราะ **ยังไม่ถูกใช้** แล้วสปายเจ้าของถูกสุ่มเปลี่ยนเป็นคนปกติ → **เกราะย้ายไปหาสปายคนใหม่ของ slot นั้น** (ตาม role ที่ได้รับใหม่)
- ดังนั้นเกราะจะ**อยู่กับสปายเสมอ** ตราบที่ยังไม่ถูกใช้ (ไม่มีวันค้างกับคนปกติ)
- 🧠 ผลต่อข้อมูล: เมื่อมีการสุ่มใหม่โดยเกราะยังไม่ถูกใช้ → **สปายคนใหม่จะรู้ว่าตัวเองมีเกราะ** (เห็นตอนดู role) ส่วนทีมยังรู้แค่ "slot A/B มีเกราะ" ไม่รู้ตัวคน → เกมขยับเล็กน้อยจากข้อมูลสำคัญนี้
- (ถ้าเกราะ **ถูกใช้ไปแล้ว** ก่อนสุ่มใหม่ → หายจากเกม สปายใหม่ไม่มีเกราะ แต่ทีมอาจยังเข้าใจว่ามีอยู่ เพราะการใช้เงียบสนิท)

## 7. ลำดับการคำนวณผลโหวต (สำคัญมากตอนสร้าง)

เมื่อหลายไอเทมโผล่ในโหวตเดียว ระบบต้องคิดตามลำดับนี้เป๊ะๆ:

```
1. นับเสียงดิบ            (รวม "โหวต 2 เสียง" เข้าไปแล้ว)
2. ใช้ "ลบ 1 เสียง"       (ลดเสียงเป้าที่เลือก)
3. ใช้ "สลับผลโหวต"       (สลับจำนวนเสียงระหว่าง 2 คน)
4. หาเกณฑ์ชนะ:
   base = round(0.72 × คนที่มา)
   ── ประมวลไอเทมหมวดเกณฑ์ "ตามลำดับที่เดินมาใช้ (turn order)" ──
   ตั้ง reduction = 0, usedR = false, usedP = false
   วนทีละการใช้:
     • ถ้าเป็น R: ถ้า (usedR || usedP) → บล็อก/เสียฟรี + เด้งข้อความ
                  ไม่งั้น → reduction = 25% ⚙️, usedR = true
     • ถ้าเป็น P: ถ้า usedP → เสียฟรี + เด้งข้อความ
                  ไม่งั้น (usedP=false):
                     ถ้า usedR == false → usedP = true (รอบล็อก R ที่จะตามมา)
                     ถ้า usedR == true  → reduction = 12% ⚙️ (อ่อนค่า), usedP = true
   threshold = round(base × (1 − reduction)), floor ≥ 2
5. เทียบ: มีใครถึงเกณฑ์ไหม → ชนะ/แพ้
6. ถ้าชนะ + ผู้ชนะ = สปายมีเกราะ → พลิกเป็น "แพ้" (เกราะทำงาน, ถูกใช้ทิ้ง)
--- ประกาศผลแพ้/ชนะ ---
7. คำนวณกองผู้ถูกโหวต = set ของคนที่ได้เสียง ≥ 1 (จากผลข้อ 3 ที่ปรับไอเทมแล้ว)
8. §4.5 ถ้า |กอง| ≥ 4 และมีสปายในกอง → ประกาศ "สปาย S จาก T คน"
9. §4.6 เปิดช่วงซื้อเบาะแส (อ้างอิงกองผู้ถูกโหวต/ไม่ถูกโหวตจากข้อ 7)
```

> **หมายเหตุการสร้าง:**
> - ลำดับนี้ต้อง deterministic และมี unit test ครอบ edge case (เสียงเท่ากันหลายคน, ลบเสียงจนติดลบ = 0, สลับกับตัวเอง, ลดเกณฑ์จนต่ำกว่า 1)
> - "กองผู้ถูกโหวต" (ข้อ 7) ใช้ผล**หลังปรับไอเทม** → สปายที่ลบเสียงตัวเองจนเหลือ 0 จะหลุดจากกอง (ตั้งใจ)
> - ✅ **ตัดสินแล้ว:** ตอนเกราะพลิกผล (ข้อ 6) เป็น**ความลับเงียบสนิท** ไม่มีการประกาศ/แจ้งเตือนใดๆ และตัวสปายก็ไม่รู้ว่าถูกใช้ (ดู §6.4)
> - ✅ **§4.5 ยังประกาศตามปกติ** ถ้าตรงเงื่อนไข (≥4 ในกอง + มีสปายในกอง) — ไม่ผูกกับเกราะ
>   - หมายเหตุความน่าจะเป็น: เคสที่เกราะทำงาน **พร้อมกับ** §4.5 ประกาศ เกิดได้แต่**ยาก** — เพราะเกราะทำงานต้องมีสปายได้ถึงเกณฑ์ (เช่น 8 เสียงจาก 11 คน) เหลือเสียงให้คนอื่นแค่ ~3 เสียง ต้องกระจายไม่ซ้ำให้กองมี ≥4 คนพอดี จึงโอกาสต่ำ (และยังแปรผันถ้ามีไอเทมโหวต 2 เสียง/ลบ/สลับ)

## 8. Flow ตลอด 1 สัปดาห์

```
จันทร์ (เริ่ม)  → สุ่มสปาย → ทุกคนเดินมาดู role → เริ่มขาย/รับเหรียญ → เล่นร้านค้า/กาชา → (อาจ) เปิดโหวต 1 ครั้ง
อังคาร–ศุกร์    → รับเหรียญตามโปรวัน → ร้านค้า/กาชา → เปิดโหวตได้วันละ 1 (ไม่เปิด = ค่าโหวตสะสมแพงขึ้น)
เสาร์–อาทิตย์   → หยุดทั้งหมด (ไม่ขาย ไม่ได้เหรียญ ไม่เปิดโหวต ค่าโหวตไม่ขยับ)
จันทร์หน้า บ่าย → เดดไลน์ตัดสิน: จับครบ = ทีมชนะ / ไม่ครบ = สปายชนะ
```

ระหว่างทาง เมื่อทายสปายคนที่ 2 ผิด → สุ่มสปายใหม่ → ดู role ใหม่ → เกมดำเนินต่อในไทม์ไลน์เดิม

## 10. 🛡️ ประเด็น "เกมพังได้ยังไง" ที่ต้องกันไว้ (Defensive)

| ความเสี่ยง | วิธีกัน |
|-----------|--------|
| state หายระหว่างสัปดาห์ (เคลียร์ cache/refresh) | persist localStorage + **export/import ไฟล์ backup** + auto-save ทุก action |
| แอดมินกดผิด | ทุก action มี **undo/แก้ไข** + log ประวัติ |
| ตั้งค่า config พัง (เกณฑ์ > คนที่มา ฯลฯ) | **validate** ค่าก่อนบันทึก + กันค่าที่เป็นไปไม่ได้ |
| คนถัดไปแอบเห็น role/โหวตคนก่อน | หน้า **"ส่งเครื่องต่อ"** คั่นทุกคน + auto-hide |
| โหวตซ้ำ/ตกหล่น | ระบบ **เช็ครายชื่อคนที่มา** วันนั้น + ติ๊กว่าใครโหวตแล้ว |
| iPad หลับ/หมุนจอ | ล็อกแนวนอน + กัน sleep ระหว่างเกม |
