# 02 — Art Direction & Image Brief (สำหรับ codex สร้างรูป)

> เป้าหมาย: ให้ codex สร้างภาพประกอบที่ **เข้ากับคอนเซปต์เดียวกันทั้งเกม**
> ทุกภาพออกแบบสำหรับ **iPad Air 5 แนวนอน** — แคนวาส logical **1180 × 820 pt**, asset จริง **@2x = 2360 × 1640 px**

---

## 1. คอนเซปต์ภาพรวม

**"สืบสวนนัวร์ แต่ตลก" (Comedic Detective Noir) — งานอาร์ตการ์ตูน**

ผสม 2 อารมณ์:
- 🔍 **สืบสวน**: แว่นขยาย, รอยนิ้วมือ, รอยเท้า, เงาสปอตไลต์, แฟ้มคดี, เครื่องหมายคำถาม, ป้ายหลักฐาน, เทปกั้นที่เกิดเหตุ
- 😂 **การ์ตูนตลก (base)**: ตัวละครหัวโต ตากลมโต สีหน้าโอเวอร์ เคลื่อนไหวเว่อร์ มุกตลกสายลับเงอะงะ ออฟฟิศวุ่นวาย

**ห้าม:** ภาพสมจริง, นัวร์มืดหม่นเกินไป, น่ากลัว/ซีเรียส — ต้อง **สดใส เป็นมิตร ขำๆ** เพราะเล่นกันในที่ทำงาน

## 2. สไตล์ภาพ (Visual Style Guide)

| องค์ประกอบ | กำหนด |
|-----------|-------|
| ลายเส้น | เส้นขอบหนา ชัด (bold outline) สไตล์การ์ตูนสมัยใหม่ |
| การลงสี | flat color + ไล่เฉดนุ่มๆ (soft cel-shading) |
| รูปทรง | โค้งมน เป็นมิตร (rounded, chunky shapes) |
| อารมณ์ | สดใส ขี้เล่น มีพลัง |
| พื้นผิว | เพิ่ม grain/paper texture บางๆ ให้ดูเป็นแฟ้มคดี |

### จานสี (Color Palette)
- **หลัก**: น้ำเงินสืบสวน `#2B3A67`, ครีมกระดาษ `#F5E9D0`, แดงหลักฐาน `#E94F37`
- **รอง**: เหลืองสปอตไลต์ `#F6C453`, เขียวมิ้นต์ `#3DCCA1`
- **กลาง/เงา**: เทาถ่าน `#3A3A3A`, ขาวนวล `#FBF7EF`
- **ทอง (เหรียญ)**: `#F2A900` → ไฮไลต์ `#FFD66B`

### ฟอนต์
- หัวเรื่อง/ดราม่า: ฟอนต์การ์ตูนหนาๆ มีบุคลิก (เช่น display rounded ตัวหนา)
- เนื้อหา/ตัวเลข: ฟอนต์อ่านง่าย ตัวหนาพอ มองจากไกลออก (เล่นเป็นกลุ่ม)
- รองรับ **ภาษาไทย** ครบ (ทดสอบสระบน-ล่าง วรรณยุกต์)

## 4. ตัวอย่าง Prompt ให้ codex สร้างรูป (อังกฤษ — โมเดลภาพมักได้ผลดีกว่า)

> ใส่ **style suffix** เดียวกันต่อท้ายทุก prompt เพื่อให้ภาพคุมโทนเป็นชุดเดียวกัน

**Style suffix (แปะท้ายทุกอัน):**
```
, cartoon detective noir style, bold clean outlines, flat colors with soft cel-shading,
rounded chunky friendly shapes, playful comedic mood, bright palette (navy #2B3A67,
cream #F5E9D0, evidence red #E94F37, spotlight yellow #F6C453), subtle paper grain
texture, professional game art, high quality, centered composition
```

ตัวอย่างราย asset:
- **role-spy**: `A goofy cartoon office spy wearing a long trench coat, fedora hat and dark sunglasses, peeking suspiciously with a sneaky grin, holding a magnifying glass [+ style suffix]`
- **role-normal**: `A cheerful cartoon office worker smiling, holding a coffee mug and a work folder, friendly pose [+ style suffix]`
- **coin**: `A shiny golden cartoon coin with an engraved magnifying-glass detective emblem, glossy highlight [+ style suffix]`
- **gacha-machine**: `A cute cartoon gacha capsule machine with blinking lights, full of colorful capsules, office setting [+ style suffix]`
- **bg-detective-room**: `Wide background of a cartoon detective investigation room, corkboard with red string connecting photos, desk lamp, evidence files, empty center space for UI, landscape 16:11 [+ style suffix]`
- **item-spy-shield**: `A premium trading card illustration of a mysterious protective shield glowing with a dark aura, spy emblem in center [+ style suffix]`

## 5. 🎬 Motion / Animation (juice)

ใส่ความมีชีวิตทุกจุด — เน้น **feedback ทันทีที่กด** และ **ดราม่าตอนประกาศผล**

### 5.1 หลักการ
- ทุกปุ่ม: กดแล้ว **เด้ง (squash & stretch)** + เงาขยับ
- ทุก transition ระหว่างหน้า: **wipe ด้วยแว่นขยาย** หรือสปอตไลต์กวาด
- ใช้ easing สนุกๆ (overshoot/bounce) ไม่ใช่ linear

### 5.2 จุดที่ต้องมี animation เด่นๆ
| จังหวะ | Animation |
|--------|-----------|
| เปิดดู role | การ์ดพลิก (flip) เผยบทบาท + ถ้าเป็นสปาย มี **ออร่าแดงลึกลับ** + คู่หูเฟดเข้า |
| ส่งเครื่องต่อ | ม่านสปอตไลต์ปิดลง → "ส่งให้คนถัดไป" |
| เปิดร้าน/กาชา | ตู้กาชา **สั่น → แคปซูลหล่น → เปิดป๊อป** + confetti |
| ได้เหรียญ | เหรียญทอง **เด้งหมุน** ลอยขึ้น + เสียงกริ๊ง |
| ลงคะแนน | บัตรลอยลงหีบ + ติ๊กถูกเด้ง |
| **ประกาศผลโหวต** | กลองรัว → ตราประทับ **กระแทกลงจอ** ("จับได้!"/"พลาด!") + จอสั่น (screen shake) |
| เกราะสายลับทำงาน | 🤫 **ไม่มีเอฟเฟกต์พิเศษ** — แสดงเป็น "แพ้" ธรรมดาเหมือนทุกครั้ง (เกราะทำงานเงียบใน backend, แยกไม่ออก) |
| ทีมชนะ/สปายชนะ | ฉากจบใหญ่ confetti / เงาสายลับหัวเราะ + สปอตไลต์ |

### 5.3 เทคนิคแนะนำ
- ใช้ **CSS transitions/keyframes + Web Animations API** (เบา ไม่ต้องไลบรารีหนัก)
- พิจารณา **Lottie** สำหรับ animation ซับซ้อน (ตู้กาชา, ตราประทับ, ฉากจบ) — export จาก After Effects ได้
- particle/confetti: ไลบรารีเล็กๆ เช่น `canvas-confetti`
- คุม performance บน iPad: หลีกเลี่ยง animate ของหนักพร้อมกันเยอะ, ใช้ `transform`/`opacity` (GPU)

## 6. 🔊 Sound Design

เสียงทำให้เกม "มีน้ำหนัก" — ต้องมี **เปิด/ปิดเสียงได้** (อยู่ในที่ทำงาน)

| เหตุการณ์ | เสียง |
|----------|-------|
| กดปุ่มทั่วไป | คลิกนุ่มๆ (pop) |
| ได้เหรียญ | เสียงเหรียญกริ๊ง/ตกกระทบ |
| เปิดร้าน/ซื้อสำเร็จ | กระดิ่งร้าน "ติ๊ง!" |
| หมุนกาชา | เสียงหมุน → "ป๊อป!" ตอนเปิด (แจ็คพอตมีแฟนแฟร์พิเศษ) |
| เริ่มโหวต | กลองตึงเครียดเบาๆ (suspense) |
| ประกาศผลชนะ | กลองรัว → ฉ่าง! (sting ดราม่า) |
| ประกาศผลแพ้ | เสียง "ตึ่งโป๊ะ" ตลกๆ |
| เกราะทำงาน | 🤫 **ใช้เสียงเดียวกับ "แพ้ปกติ"** — ห้ามมีเสียงพิเศษ (ไม่งั้นจะเดาออกว่ามีเกราะ) |
| ทีมชนะ | แฟนแฟร์ชัยชนะสดใส |
| สปายชนะ | หัวเราะร้ายๆ + ดนตรีลึกลับ |

- มี **background music** เบาๆ โทนสืบสวนขี้เล่น (jazzy detective) เปิด/ปิดได้
- ไฟล์เสียง: ใช้ `.mp3`/`.ogg` ขนาดเล็ก, preload, เล่นผ่าน Web Audio API
- ⚠️ iOS Safari: เสียงเล่นได้หลัง **user interaction แรก** เท่านั้น → มีปุ่ม "เริ่มเกม" ปลดล็อก audio
