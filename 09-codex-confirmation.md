## ⚠️ ดีเทลที่ต้องทำให้ครบ (ในไฟล์ codex เขียนรวมๆ — กางกฎไว้ให้แล้ว)

### 1) เบาะแสหลังโหวต (§4.6) = **งานกลุ่ม**
- เป็นงานกลุ่ม **เหมือนการเปิดโหวต**: ทีมรวมเหรียญซื้อร่วมกัน
- **ราคา = 1/4 ของค่าเปิดโหวตรอบนี้** (ปัดลง)
- ทำได้ **1 ครั้งต่อรอบโหวต** (ทั้งทีม ไม่ใช่ต่อคน)
- **ผลเปิดเผยให้ทุกคนเห็นบนจอใหญ่**
- เลือก 1 ใน 2:
  - **(ก) ดูรูปคนที่ถูกโหวต**: ถ้าผู้ถูกโหวต **≥ 3 คน** → สุ่มโชว์ 1 คน; ถ้า **< 3 คน** → จ่ายฟรี ไม่ได้อะไร
  - **(ข) ดูรูปคนที่ไม่ถูกโหวต**: สุ่มโชว์ **สูงสุด 4 คน** (มีน้อยกว่าก็โชว์เท่าที่มี)
- "กองผู้ถูกโหวต" คิด **หลังปรับไอเทม/เอฟเฟกต์ทุกอย่างแล้ว**

### 2) ประกาศจำนวนสปายในกอง (§4.5)
- ทำงานหลังประกาศผล (ทั้งแพ้และชนะ)
- เงื่อนไข: **ผู้ถูกโหวต ≥ 4 คน** (หลังปรับไอเทม) **และ** มีสปายในกองอย่างน้อย 1 คน
- ประกาศ *"มีสายลับ S คน จากผู้ถูกโหวต T คน"* — **ไม่บอกชื่อ ไม่บอกจำนวนเสียง**
- ถ้าไม่มีสปายในกอง → **เงียบ**
- ยังประกาศตามปกติแม้เกราะพลิกผล (default) — เคสนี้เกิดยากมากอยู่แล้ว
- ⚠️ เกณฑ์ของ §4.5 (≥4) **แยกจาก** เบาะแส (ก) (≥3) อย่าใช้ปนกัน

### 3) ข้อความเตือนไอเทมหมวดเกณฑ์ (R/P)
- ตอนกดใช้ไอเทมที่ซ้ำ/ถูกบล็อก → เด้งข้อความ *"มีคนใช้ไอเทมหมวดเกณฑ์ไปแล้ว"*
- **ห้ามบอกว่าเป็น R หรือ P** (กันข้อมูลรั่ว — สปายต้องคุยกันเอง)
- กฎตัดสิน (ตาม §7 ข้อ 4):
  - **R**: ถ้ามีหมวดเกณฑ์ใดถูกใช้ก่อนหน้า (R หรือ P) → R ถูกบล็อก/เสียฟรี + เด้งข้อความ; ไม่งั้น reduction = 25%
  - **P**: ถ้ามี P ถูกใช้ก่อนหน้า → เสียฟรี + เด้งข้อความ; ไม่งั้น → ถ้ายังไม่มี R = รอบล็อก R ที่จะตามมา / ถ้ามี R แล้ว = อ่อนค่าเหลือ 12%
  - threshold = round(base × (1 − reduction)), ขั้นต่ำ ≥ 2

### 4) คืนเหรียญ 1/4 (โหวตชนะ-โดนคนบริสุทธิ์)
- คืน **1/4 ของกองที่จ่าย** (ปัดลง)
- **จำนวนผู้รับ < จำนวนคนที่มา** (ต้องมีคนไม่ได้คืน)
- ทีมเลือก **1 คนเป็นผู้ถือ** → คนนั้นจัดสรรเอง

### 5) เกณฑ์ชนะ & ค่าเปิดโหวต (scale ตามคนที่มา)
- **เกณฑ์ชนะ = round(0.72 × จำนวนคนที่มาวันนั้น)** (config ปรับได้)
- **ค่าเปิดโหวต** scale ตามจำนวนคนที่มา + **+50% ต่อวันทำงานที่ข้ามโดยไม่เปิดโหวต** (สะสม, รีเซ็ตเมื่อเปิด)
- ตัวคูณกาชา **×1.5 / ×0.5** ใช้กับค่าเปิดโหวต "ครั้งถัดไป" 1 ครั้ง
- เปิดโหวตได้ **วันละ 1 ครั้ง**
- คนลาวันนั้น: ถ้าทีมชนะ **ไม่ได้รับส่วนแบ่ง**

### 6) ตัดสินตอนเดดไลน์
- เดดไลน์ = แอดมินกด final day เอง
- ตอนนั้น: ถ้าทีม **ยังจับสปายไม่ครบ 2 คน** → **สายลับชนะทั้งคู่** (เล่น end-scene ฝั่งสปาย)
- ถ้าทีมจับครบก่อนหน้าแล้ว → จบเป็นทีมชนะไปก่อนแล้ว

## 🔒 จุดที่ต้องเคาะ/ระวัง (decision + ความเสี่ยง)

### A) รูปจริงพนักงาน + โฮสต์สาธารณะ
- รูปเป็นลิงก์ Google Drive (`lh3.googleusercontent.com/d/...`) → ต้องตั้งแชร์ **"Anyone with the link / ทุกคนที่มีลิงก์"** ทุกไฟล์ ไม่งั้นรูปไม่ขึ้น (เด้ง fallback)
- หน้าเว็บบน GitHub Pages **เปิดได้ด้วยลิงก์สาธารณะ** → รูปพนักงานจะอยู่บน URL ที่คนนอกเข้าถึงได้
- ✅ ปลอดภัย: state เก็บในเครื่อง คนนอกแก้เกมจริงบน iPad เราไม่ได้ (ไม่มี PIN = โอเค)
- ⚠️ ต้องเคาะ: รับได้ไหมที่รูปพนักงานอยู่บนลิงก์สาธารณะ (เรื่องความเป็นส่วนตัว) — ถ้าไม่ ค่อยพิจารณาโฮสต์แบบจำกัดสิทธิ์/ใส่ token

### B) ความทนของ state (เกมยาว 1 สัปดาห์)
- แนะนำเก็บ state ด้วย **IndexedDB** (ทนกว่า localStorage ที่ iOS Safari อาจ evict)
- บังคับ/เตือนให้ **export backup ทุกวัน** กันเหนียว
- กัน iPad sleep ระหว่างเกม + เตือนก่อน refresh/ปิดแท็บ
