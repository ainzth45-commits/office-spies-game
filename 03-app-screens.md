# 03 — สเปกหน้าจอ (App Screens) — iPad Air 5 แนวนอน

> อุปกรณ์เป้าหมาย: **iPad Air 5 (M1) แนวนอน**
> - ความละเอียดจริง: **2360 × 1640 px** (@2x)
> - แคนวาสออกแบบ (logical): **1180 × 820 pt** · อัตราส่วน ~1.44:1
> - เผื่อ safe area บน-ล่าง ~24pt, ซ้าย-ขวา ~32pt
> - **ล็อกแนวนอน** ตลอดเกม · ปุ่ม/ตัวอักษรใหญ่พอมองจากกลุ่ม (touch target ≥ 60pt)

แอปทำงานเป็น **Single-Page Web App** รันใน Safari/PWA บน iPad เครื่องเดียว
มี 2 บริบทผู้ใช้: **แอดมิน** (คุมเกม) และ **ผู้เล่น** (เดินมาทำรายการทีละคน)

---

## 0. แผนผังหน้าจอ (Screen Map)

```
[Boot/Unlock] → [Home / Admin Console]
                   ├── Game Setup (ตั้งค่าผู้เล่น + config)
                   ├── Role Reveal Flow (ดู role ทีละคน)
                   ├── Shop (3 หมวด)
                   ├── Gacha
                   ├── Vote Flow (เปิดโหวต → ลงคะแนนลับ → ประกาศผล)
                   ├── Guess-Second-Spy (ตอนโหวตโดนสปาย)
                   ├── Day/State Dashboard (วันนี้วันไหน, ค่าโหวตปัจจุบัน, สถานะเกราะ)
                   └── End Game (ทีมชนะ / สปายชนะ)
```

---

## 1. Boot / Unlock
- ปุ่มใหญ่ **"เริ่มเกม"** กลางจอ (โลโก้ + แว่นขยาย)
- ทำหน้าที่ **ปลดล็อก audio** ของ iOS (ต้องมี user interaction ก่อนเล่นเสียง)
- โหลด state จาก localStorage ถ้ามีเกมค้างอยู่ → ถาม"เล่นต่อ / เริ่มใหม่"

## 2. Home / Admin Console
แดชบอร์ดหลักของแอดมิน — โชว์สถานะเกมแบบรวดเดียว:
- 📅 **วันที่เท่าไหร่ของสัปดาห์** (จ.–จ.หน้า) + นับถอยหลังถึงเดดไลน์
- 👥 จำนวนคนที่มาวันนี้ (ติ๊กเช็คชื่อ คนลา)
- 🗳️ **ค่าเปิดโหวตปัจจุบัน** (โชว์ว่าสะสมแพงขึ้นเท่าไหร่) + เปิดโหวตวันนี้แล้วหรือยัง
- 🛡️ สถานะเกราะสายลับ (มีในเกมไหม — โชว์แค่ slot A/B ตามที่ประกาศ)
- ปุ่มลัดไปทุกฟังก์ชัน: Setup · Role · Shop · Gacha · Vote
- ปุ่ม **Backup/Export** + **เปิด/ปิดเสียง**

## 3. Game Setup
- เพิ่ม/แก้รายชื่อผู้เล่น + เลือก avatar
- เลือกจำนวนสปาย (default 2)
- เข้าหน้า **Config** (ดู [04-config-defaults.md](04-config-defaults.md)) — ปรับตัวเลขทั้งหมด มี validate กันค่าพัง
- ปุ่ม **"สุ่มบทบาท"** → เข้าสู่ Role Reveal Flow

## 4. Role Reveal Flow (ดู role ทีละคน) 🔒
ใช้ทั้งตอนเริ่มเกม + ตอนสุ่มสปายใหม่
1. หน้า **เลือกชื่อตัวเอง** (ผู้เล่นกดชื่อตัวเอง)
2. ปุ่ม **"แตะค้างเพื่อดูบทบาท"** → การ์ดพลิกเผย role
   - ผู้เล่นปกติ: "คุณคือผู้เล่นปกติ"
   - สปาย: "คุณคือสายลับ A/B" + โชว์คู่หู (ออร่าแดง + เสียงลึกลับ)
3. ปล่อยมือ/กดเสร็จ → **หน้าม่านปิด "ส่งเครื่องให้คนถัดไป"** (bg-role-cover)
4. ระบบติ๊กว่าใครดูแล้ว จนครบทุกคน → กลับ Home

> 🔒 กันข้อมูลรั่ว: ไม่มีทางย้อนดู role คนอื่น, auto-hide หลังปล่อยมือ, คนถัดไปเริ่มจากหน้าม่านเสมอ

## 5. ร้านไอเทม (ซื้อลับ)
- ผู้เล่นเดินมา → แอดมินเลือก "ใครกำลังซื้อ" (เพื่อผูกกับกระเป๋าลับ)
- **ไอเทมโหวต (ลับ)**: หน้าซื้อ **ส่วนตัว** (คนอื่นไม่เห็นว่าซื้ออะไร — โหวต2/ลบ/สลับ/R/P) → เก็บเข้ากระเป๋าลับของคนนั้น → จบด้วยหน้าม่าน
- *(ไม่มีการขายโจทย์เชาว์ในร้าน — โจทย์มาจากกาชาเท่านั้น)*

## 6. Gacha (10 ผล)
- ตู้กาชากลางจอ → ปุ่ม "หมุน" (สั่น → แคปซูลหล่น → เปิดป๊อป + confetti)
- ประกาศผลเปิดเผย (ยกเว้นเกราะ: ประกาศแค่ slot A/B)
- ถ้าออกเกราะ → บันทึก (ผูก slot) + ป้ายประกาศ "สปาย A/B ได้รับเกราะ!"
- ถ้าออก 🎁 ไอเทม → ใส่กระเป๋าผู้หมุน + ประกาศชนิด
- ถ้าออก 🧠 **โจทย์เชาว์** → เปิดหน้าโจทย์ 2 ตัวเลือกทันที → ทีมปรึกษา → ผู้หมุนกดตอบ → เฉลย + ประกาศผล (ผู้หมุนได้/ทุกคนเสีย)

## 7. Vote Flow ⭐
**7.1 เปิดโหวต**
- แอดมินกด "เปิดโหวตวันนี้" → โชว์ค่าเหรียญที่ต้องรวม (ตามวันที่สะสม) → ยืนยันว่าจ่ายครบ

**7.2 ลงคะแนนลับ** (ห้ามปรึกษากัน)
- วนทีละคน: เลือกชื่อตัวเอง → หน้าโหวตส่วนตัว: **โชว์กระเป๋าไอเทมของตัวเอง** (รวมเกราะถ้ามี) + เลือกเป้า + (ถ้ามีไอเทมโหวต) เลือกใช้ไอเทม → ยืนยัน → หน้าม่าน
- 🤫 หน้านี้คือ**จุดเดียว**ที่สปายจะรู้ว่าเกราะยังอยู่/หายไปแล้ว (ถ้ารอบก่อนถูกใช้เงียบๆ รอบนี้จะเห็นว่าหาย)
- ระบบติ๊กคนที่โหวตแล้ว จนครบคนที่มา

**7.3 ประกาศผล** (จอใหญ่ ดราม่า — ไม่โชว์ตัวเลข)
- คำนวณตามลำดับใน [01 §7] → กลองรัว → ตราประทับกระแทก
  - **แพ้**: "พลาด!" เหรียญคืนซุป
  - **ชนะ-โดนบริสุทธิ์**: โชว์คนโดน → เข้าหน้าจัดสรรคืน 1/4
  - **ชนะ-โดนสปาย**: → เข้าหน้า Guess-Second-Spy
  - **เกราะทำงาน**: 🤫 แสดงผลเป็น "พลาด!" ธรรมดา **เหมือนแพ้ปกติทุกประการ** — ไม่มีเอฟเฟกต์/เสียง/ข้อความพิเศษ (แม้แต่สปายเจ้าของเกราะก็แยกไม่ออก)

**7.4 ประกาศจำนวนสปายในกอง (§4.5)** — หลังผลแพ้/ชนะ
- ถ้าผู้ถูกโหวต ≥ 4 และมีสปายในกอง → ป้ายดราม่า *"รอบนี้มีสายลับ S คน จากผู้ถูกโหวต T คน"* (แว่นขยาย ไม่บอกชื่อ/เสียง)
- ไม่เข้าเงื่อนไข → ข้ามเงียบ

**7.5 ช่วงซื้อเบาะแส (§4.6)** 🃏 — **งานกลุ่ม (เหมือนเปิดโหวต)**
- ทีมตกลงรวมเหรียญ → แอดมินยืนยันจ่าย 1/4 ค่าโหวตรอบนี้ → เลือก 1 ใน 2:
  - (ก) ดูรูป "คนที่ถูกโหวต" → สุ่มโชว์ 1 คน (ถ้า **<3 คน** = จ่ายฟรี แจ้งว่าเสี่ยงแล้วพลาด)
  - (ข) ดูรูป "คนที่ไม่ถูกโหวต" → สุ่มโชว์สูงสุด 4 คน
- โชว์ผลเป็น **การ์ดรูป/หมายจับ บนจอใหญ่ เห็นกันทั้งทีม** · ทำได้ **1 ครั้ง/รอบโหวต**

## 8. Guess-Second-Spy
- ทีมปรึกษากันได้ → เลือก 1 ชื่อ → ยืนยัน (ตอบได้ครั้งเดียว)
  - **ถูก** → ฉากทีมชนะ
  - **ผิด** → แจ้ง "ยังไม่ใช่!" → สุ่มสปายใหม่ → เข้า Role Reveal Flow อีกรอบ

## 9. หน้าจัดสรรเหรียญคืน (1/4)
- โชว์จำนวนที่คืน (น้อยกว่าจำนวนคน) → เลือก 1 คนเป็นผู้ถือ → คนนั้นจัดสรร → บันทึก

## 10. End Game
- **ทีมชนะ**: confetti + สปอตไลต์ + ประกาศผู้ชนะ (รางวัลพูดปากเปล่า)
- **สปายชนะ** (หมดเวลา): เผยตัวสายลับ 2 คน + เงาหัวเราะร้ายๆ
- ปุ่ม "เริ่มเกมใหม่" (เคลียร์ state หลังยืนยัน + เก็บ backup ไว้)

---

## 11. องค์ประกอบ UI ที่ใช้ซ้ำ (Shared Components)
- **ม่านส่งเครื่อง** (Hand-off Cover) — คั่นทุก action ที่เป็นความลับ
- **Toast/ป้ายประกาศ** — แจ้งผลกลางจอ
- **แถบสถานะวัน** — มุมจอ โชว์วัน/เดดไลน์/ค่าโหวต
- **ปุ่มเสียง** + **ปุ่ม backup** — เข้าถึงได้ทุกหน้า

## 12. ข้อกำหนดเชิงเทคนิคของ UI
- ออกแบบ **landscape-first** ล็อกแนวนอน (กันหมุนเป็นแนวตั้ง)
- กัน iPad sleep ระหว่างเกม (Wake Lock API ถ้าได้ / fallback แจ้งให้ตั้ง auto-lock = Never)
- กันเผลอ refresh/ย้อน (เตือนก่อนออก) — state ต้อง persist อยู่แล้ว
- ทุกหน้ารองรับ **เปิด/ปิดเสียง** + แสดง animation ตาม [02 §5]
